# Move Selector

Sometimes you may have an issue where your framework does not allow you to modifying HTML in optimal places for Estimtest. A good example of this is the [`vite-vue` example](https://github.com/KyleSmith0905/estimtest/tree/main/examples/vite-vue).

Estimtest (and all of it's helpers) is able to transfer itself to a better location if the property `move-selector` is provided. The Estimtest component will move itself when the user accepts the prompt to begin testing.

## Example

In this example, pretend we do not have access to write HTML outside of `main`. We can specify our component to move up by setting the `move-selector` attribute to be the parent of where you want to move the component. For instance, the example provided will move the Estimtest component one level up.
